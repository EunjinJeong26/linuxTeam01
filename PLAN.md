### 3.4 자동 데이터 수집 메커니즘

**(1) CPU·메모리 사용량 ✅**

* 데이터 단위: **현재 시점의 순간 상태** (평균·누적 아님) — 2026-05-11 확정
* 수집 소스: `/proc/stat`, `/proc/meminfo` 파싱 (Linux 표준)

**(2) git 커밋 수집 ✅ (2026-05-14 확정)**

* **수집 방식:** 로컬 파일 시스템에서 `.git` 디렉터리를 탐색하여 해당 저장소의 커밋 내역을 수집한다. GitHub API 연동은 채택하지 않는다.
* **탐색 범위 (A안 채택):** 사용자가 명시적으로 등록한 로컬 디렉터리만 스캔한다. 등록된 디렉터리의 **하위 트리 전체**를 재귀 탐색하여 `.git` 디렉터리를 찾는다.
* **`.git` 중복 발견 시 처리:** 등록 범위 내에 `.git` 디렉터리가 2개 이상 발견된 경우, **가장 최근 커밋이 있는 저장소를 우선** 사용한다.

### 3.5 알림 메커니즘 ✅

**백그라운드 데몬 프로세스가 cron 기반으로 알림을 발생시킨다 (2026-05-11 확정).**

* 데몬 등록·실행 방식의 상세는 섹션 4.5 참조
* 알림 노출 채널: **터미널 벨** (ASCII `\a` / BEL)

### 3.6 CLI 명령어 명세 ✅ (2026-05-24 구조 반영)

## 4. 아키텍처 결정

### 4.1 통신 모델: 중앙 서버 (Client-Server) ✅

**결정 사유:** P2P의 NAT 통과 복잡도를 회피하고 "실사용 가능성" 채점 기준(오프라인 시 데이터 흐름 유지)을 충족.

### 4.2 통신 방식: 하이브리드 (REST + WebSocket) ✅

* **REST:** 게시물 작성·조회, 그룹·사용자 관리 등 요청-응답 작업
* **WebSocket:** 알림 푸시, 채팅·반응의 실시간 반영

### 4.3 데이터 저장 ✅

* **서버 DBMS: SQLite** (2026-05-14 확정)
* **클라이언트 로컬 캐시: SQLite** (2026-05-24 확정) — `.txt`/CSV 방식 대신 채택
* 구성·운영이 단순하고 학기 단위 산출물에 적합. 클라이언트·서버 모두 동일 DBMS를 사용하여 스키마 설계 일관성 확보, 데몬·CLI 동시 접근 시 파일 손상 방지(ACID 보장).

### 4.4 인증·사용자 식별 ✅

* **인증 방식:** 사용자명 + 비밀번호 → 서버가 **JWT** 발급
* 세션 유지 방식: 한 번 로그인하면 토큰을 로컬 클라이언트에 저장하여 매 요청마다 헤더에 첨부.

### 4.5 백그라운드 데몬 ✅

**systemd 서비스로 등록하여 운영한다.**

* **데몬·CLI 통신 채널:** **Unix Domain Socket** 사용
* **데몬 실행 권한:** **root 권한** (권한 문제 없이 동작 시 현재 사용자 권한 폴백)

### 4.6 디렉토리 및 로컬 저장소 구조 설계 ✅ (2026-05-24 구조 반영)

├── cache.db    - SQLite 로컬 캐시 (teams, members, logs, git_dirs 테이블)

CREATE TABLE git_dirs (
    path       TEXT PRIMARY KEY,
    added_at   TEXT NOT NULL
);

-- 팀 정보 캐시 (서버 동기화)
CREATE TABLE teams (
    team_id     INTEGER PRIMARY KEY,
    team_name   TEXT NOT NULL,
    invite_code TEXT NOT NULL,
    created_at  TEXT NOT NULL
);

-- 팀 멤버십 캐시 (서버 동기화)
CREATE TABLE members (
    team_id   INTEGER NOT NULL,
    username  TEXT    NOT NULL,
    role      TEXT    NOT NULL,
    joined_at TEXT    NOT NULL,
    PRIMARY KEY (team_id, username)
);

-- 로그 캐시 (서버 동기화 및 오프라인 열람)
CREATE TABLE logs (
    log_id          INTEGER PRIMARY KEY AUTOINCREMENT,
    username        TEXT    NOT NULL,
    team_id         INTEGER NOT NULL,
    session_date    TEXT    NOT NULL,  -- YYYY-MM-DD
    session_hour    INTEGER NOT NULL,  -- 0~23
    cpu_usage       REAL,
    mem_usage       REAL,
    git_commit_hash TEXT,
    git_commit_msg  TEXT,
    comment         TEXT,
    posted_at       TEXT    NOT NULL
);

**(1) C 클라이언트 ✅:** `libcurl` (REST 통신), `cJSON` (JSON 파싱), `sqlite3` (로컬 캐시 DB), `ncurses` (TUI 도입 시 선택)
**(2) 서버 측 ⬜:** DB 드라이버, WebSocket, 인증 라이브러리 세부 선정 필요
